Agile Brain Breaks

Five short, hands‑on activities (5–8 minutes each) to apply Agile this week. Use during transitions to reinforce habits.
Mon
Daily Scrum Micro
Practice concise status, surface impediments, and re‑plan.
Do: Round‑robin: Yesterday / Today / Blockers.
  1. Each person has 30–45s to answer: What did I finish? What will I do next? Any blockers?
  2. Convert blockers into concrete impediments. Assign an owner to remove each.
  3. Re‑order the top 3 tasks on your team board if priorities shifted.
5–6 min Applies: Scrum, Transparency
Alternate: Stand‑up Remix
  1. Use only outcomes: “A thing that is now done is…”, “A thing that will be done is…”.
  2. If something appears twice in a row, mark it as a potential risk and swarm briefly.
5 minFocus: Outcomes
06:00
Tue
User Story Slicing Drill
Split a large story into vertical slices with clear acceptance criteria.
Do: Take the big story “Enable account login” and slice.
  1. Write 3–5 thin slices that each deliver user‑visible value (e.g., “Login with code only”, “Show errors”).
  2. For one slice, add 2–3 acceptance criteria (Given/When/Then).
  3. Order slices by value/risk and pick the first one to implement next.
7–8 min Applies: Story Mapping, DoR
Alternate: INVEST Check
  1. Take any current story and rate it on INVEST (Independent, Negotiable, Valuable, Estimable, Small, Testable).
  2. Fix the weakest letter with a quick edit to the story.
6–7 minFocus: Story Quality
08:00
Wed
Estimation Lightning (Planning Poker)
Estimate quickly to spot uncertainty and risk.
Do: Pick 3 backlog items and estimate with T‑shirt sizes (XS/S/M/L/XL).
  1. Silent pick → reveal together. If mismatch, one sentence from high/low to explain.
  2. Re‑estimate once. If still split, mark as “needs spike” or split the story.
6–7 minApplies: Relative Estimation
Alternate: Risk Radar
  1. For one story, list top 3 risks. Tag each as Unknowns, Dependencies, or Complexity.
  2. Pick 1 risk and define a 1‑hour spike to reduce it.
5–6 minFocus: Risk First
07:00
Thu
Kanban Flow Game (WIP 2)
Limit work‑in‑progress and expose bottlenecks.
Do: On your board (To Do → Doing → Review → Done), set WIP=2 for “Doing”.
  1. Move cards respecting WIP. If blocked, add a “⛔ Impediment” tag with cause.
  2. Identify the bottleneck column; agree on one change to speed flow (e.g., pair on Review).
6–7 minApplies: Flow, WIP Limits
Alternate: Definition of Done (DoD) Sprint
  1. Draft a 5‑item DoD for your current story (e.g., tests, review, docs, demoable).
  2. Apply it now to the top in‑progress item.
5–6 minFocus: Quality
07:00
Fri
Retro Snap + 1 Experiment
Reflect quickly and commit to one improvement.
Do: Start / Stop / Continue with dot‑vote.
  1. Each person adds 1 sticky under Start, Stop, Continue.
  2. Each gets 2 votes. Highest wins → define a small experiment for next week (owner + success criteria).
7–8 minApplies: Kaizen
Alternate: MoSCoW Prioritization
  1. Take 6–8 current tasks and sort into Must, Should, Could, Won’t.
  2. Move top “Must” to the very next sprint/iteration.
6–7 minFocus: Value First
08:00
How to Use
  • Pick today’s card. Click “Swap Activity” if the alternate fits better.
  • Press “Start Timer”. Keep it short and outcome‑focused.
  • Capture one tangible change (e.g., updated order, DoD, owner on impediment).
Tip: Repeat any favorite on future weeks; consistency builds habits.
